🛠️ The Zen of Tegar (Software Design Philosophy)
Readable Code Over Clever Tricks: Kode adalah tentang komunikasi antar manusia, komputer hanya menjalankannya.
Local-First, Cloud-Option: Keamanan data adalah mutlak. Jalankan sistem secara lokal jika memungkinkan.
Automate Everything (Within Reason): Jika Anda harus melakukan suatu tugas secara manual lebih dari 3 kali, buatkan script/agen otonom untuk tugas itu.
Crash-Safe Integrity: Gunakan format penyimpanan yang aman dari kerusakan listrik (seperti SQLite WAL mode).
Zero-Trust Input: Selalu asumsikan data input atau sesi luar tidak terpercaya sebelum diverifikasi.
Async By Default: Hindari blocking I/O pada alur kerja utama aplikasi.
Telemetry & Metrik: Anda tidak bisa mengoptimalkan performa sesuatu yang tidak Anda ukur (TTFT & token tracking).
Modular Components: Pisahkan struktur sistem secara tegas (plan, secure, build, deploy).
Secure Configuration: Hindari hardcoded keys. Gunakan proteksi penyimpanan kunci lokal (seperti OS-level DPAPI).
Midnight Debugging: Masalah yang paling rumit seringkali terpecahkan saat dunia sedang tidur.
